# 告警视听助手 — UI 优化方案

> 文档版本：2026-06-23 **v5**（MOSS 缓存拼合 + 热词管理 + 多 Tab 工作台）  
> 适用项目：`alarm_speech`（Python 3.9 + Flet 0.28）  
> 关联方案：[MOSS语音缓存与拼合方案.md](./MOSS语音缓存与拼合方案.md)

---

## 一、设计原则（延续 v4 视觉，扩展能力）

### 1.1 保持不变的风格（用户已认可）

| 元素 | 规范 | 说明 |
|------|------|------|
| 深色底 | `#0f1117` | 页面背景 |
| 卡片面 | `#181c24` / `#1f2530` | 侧栏卡片、输入框底 |
| 边框 | `#2a3140` | 细线分隔，不厚重 |
| 主强调 | `#5b9cf5` | 按钮、链接、徽章 |
| 成功 / 警告 | `#6ecf8a` / `#f0b45c` | CUDA 就绪、缓存命中、告警 |
| 顶栏 + 色条卡片 | 左侧 4px 色条 + 图标标题 | 连接=蓝、监控=绿、缓存=紫（新增） |
| 终端装饰 | 三色圆点 + Consolas 绿字 | 「数据终端」辨识度 |
| 控件 | 高 38px、圆角 8px、字号 12 | 与当前 `flet_demo.py` 一致 |

**原则：新功能用「新 Tab / 新卡片 / 新徽章」承载，不推翻现有侧栏布局。**

### 1.2 新增设计目标（v5）

1. **缓存可感知**：用户能看见「有没有开拼合、命中了几段、快了多少」。
2. **热词可管理**：固定词、历史热度词、手动添加词，列表可搜、可预热、可试听。
3. **日志可追溯**：合成、缓存、拼合、预热分通道落盘，界面一键查看。
4. **交互可反馈**：预热进度条、命中率趋势、未命中时明确提示「正在合成第 2/4 段」。

## 四、本地目录与文件设计

> 与 [MOSS语音缓存与拼合方案.md](./MOSS语音缓存与拼合方案.md) §5.2 对齐，并补充 **热词 / 统计 / 日志** 文件，便于 UI 读写。

### 4.1 目录树（项目根 `alarm_speech/`）

```
alarm_speech/
├── cache/                          # 语音缓存根（建议 .gitignore）
│   ├── moss/                       # 按引擎分子目录
│   │   └── Junhao/                 # 按 MOSS 音色
│   │       ├── 端口.wav
│   │       ├── 联通.wav
│   │       ├── 中断.wav
│   │       └── B05柜.wav
│   ├── index.json                  # 片段索引（核心）
│   ├── hotwords.json               # 热词库（核心）
│   ├── stats.json                  # 命中率等滚动统计
│   ├── warmup_job.json             # 当前/上次预热任务状态
│   └── meta.json                   # 版本、引擎、采样率、创建时间
├── logs/
│   ├── app.log                     # 已有：应用总日志
│   ├── error.log                   # 已有：异常堆栈
│   ├── moss_daemon.log             # 已有：守护进程
│   └── tts_cache.log               # 新增：缓存/拼合专用（见 §4.4）
└── config.local.json               # UI 选项持久化（见 §4.5）
```

### 4.2 `cache/index.json`（片段索引）

每条缓存片段一条记录，供 UI 列表与 LRU 淘汰。

```json
{
  "version": 1,
  "engine": "moss",
  "voice": "Junhao",
  "sample_rate": 24000,
  "entries": {
    "a1b2c3…": {
      "text": "端口",
      "slot": "keyword_port",
      "tier": "fixed",
      "path": "cache/moss/Junhao/端口.wav",
      "duration_ms": 420,
      "file_size": 20480,
      "hit_count": 128,
      "miss_count": 2,
      "last_used_at": "2026-06-23T22:24:42",
      "created_at": "2026-06-20T10:00:00"
    }
  }
}
```

| 字段 | UI 用途 |
|------|---------|
| `text` | 热词列表主列 |
| `slot` | 槽位标签（机柜/设备/状态/固定词） |
| `tier` | `fixed` / `hot` / `user` 筛选 |
| `hit_count` | 排序、展示「热度」 |
| `path` | 试听、打开文件夹 |

**cache_key**：`hash(engine + voice + text_normalized)`，与 MOSS 方案一致。

### 4.3 `cache/hotwords.json`（热词库）

热词 = **计划缓存或已缓存的文本片段**，分三层来源。

```json
{
  "version": 1,
  "fixed_words": ["端口", "联通", "中断", "恢复", "告警"],
  "user_words": ["B05柜", "NE20E杠1"],
  "hot_from_history": [
    { "text": "C12柜", "count": 45, "last_seen": "2026-06-23" }
  ],
  "blocked": [],
  "settings": {
    "max_entries": 500,
    "max_disk_mb": 500,
    "auto_add_on_miss": true
  }
}
```

| 层级 | 来源 | 管理方式 |
|------|------|----------|
| **Tier-0 固定词** | 内置词表 | 只读展示，可「一键预热」 |
| **Tier-1 历史热度** | 拉历史告警统计 | 「从历史分析」按钮生成 |
| **Tier-2 用户自定义** | 用户手动添加 | 输入框 + 添加，可删除 |

`blocked`：用户禁止缓存的词（隐私/测试用）。

### 4.4 `logs/tts_cache.log`（缓存专用日志）

单行一条，便于终端 Tab 过滤着色。

```
2026-06-23 22:24:42 INFO  STITCH text="B05柜，NE20E杠1，端口，联通" hits=4/4 miss=0 elapsed_ms=280
2026-06-23 22:24:40 INFO  MISS  text="联通" synthesizing…
2026-06-23 22:24:38 INFO  HIT   text="端口" path=cache/moss/Junhao/端口.wav
2026-06-23 22:20:00 INFO  WARMUP progress=37/250 tier=fixed
```

UI「落盘日志」Tab 提供子筛选：**全部 / 应用 / 缓存 / MOSS 守护进程**。

### 4.5 `config.local.json` 扩展字段

```json
{
  "alarm_speech": {
    "poll_interval_seconds": 30,
    "speech_text_rule": "auto",
    "speech_play_policy": "once",
    "speech_play_policy_params": { "cooldown_seconds": 120, "repeat_n": 3 },
    "tts_backend": "moss",
    "moss_execution_provider": "cuda",
    "moss_cpu_threads_preset": "reserve1",
    "cache": {
      "enabled": true,
      "stitch_silence_ms": 80,
      "fallback_whole_sentence": true,
      "auto_warmup_on_start": false
    }
  }
}
```

### 5.3 状态与颜色（与 v4 色板一致）

| 状态 | 徽章 | 色 |
|------|------|-----|
| 已缓存 | ● 已缓存 | 绿 `#6ecf8a` |
| 待合成 | ○ 待合成 | 灰 `#8b93a7` |
| 合成中 | ⟳ 合成中 | 蓝 `#5b9cf5` |
| 失败 | ✕ 失败 | 红 `#e85d5d` |

### 5.4 核心交互流程

**流程 A — 首次使用**

1. 用户选 MOSS 本地 → 左栏出现「语音缓存」卡。  
2. 提示：「建议先预热固定词（约 1 分钟）」→ 点 **预热固定词**。  
3. 进度条走完 → 命中率徽章显示 → 试听同一句应明显变快。  

**流程 B — 从历史学习**

1. 先 **拉取历史 100 条**（已有按钮）。  
2. 缓存 Tab → **从历史分析热词** → 弹出预览：「将添加 23 个设备名片段」。  
3. 确认 → 写入 `hotwords.json` → 可选立即 **批量预热**。  

**流程 C — 运行时未命中**

1. 告警触发拼合 → 终端显示：`片段 3/4 命中，正在合成「联通」`。  
2. 合成完成 → 自动写入 `index.json` → 热词表该行变为「已缓存」。  
3. `tts_cache.log` 写 `MISS` + `HIT` 记录。  

**流程 D — 试听拆分预览**

1. 用户在「手动添加」输入：`B05柜，NE20E杠1，端口，联通`。  
2. 点 **试拆分** → 展示 4 个片段及各自缓存状态（只读预览）。

## 六、缓存与拼合 — 界面呈现规范

### 6.1 三层呈现（由轻到重）

| 层级 | 位置 | 内容 | 更新时机 |
|------|------|------|----------|
| **L1 徽章** | 顶栏 | `缓存 82%` 或 `拼合开` | 每次播报后滚动统计 |
| **L2 摘要** | 左栏缓存卡 | 已缓存条数、磁盘占用、上次预热时间 | 预热完成 / 定时刷新 |
| **L3 详情** | 语音缓存 Tab | 表格、日志、预热进度 | 用户主动打开 |

### 6.2 数据终端中的合成摘要行（B.4 扩展）

每条需播报的告警，在事件下增加 **固定格式** 子行（Consolas，缩进 2 空格）：

```
[22:24:42] 新增告警 | B05_NE20E-1 | 端口Down
  文本: 端口表 → 「B05柜，NE20E杠1，端口，中断」
  策略: 仅播一次
  合成: 拼合 4/4 命中 | 280ms | 静音 80ms
```

未全开命中时：

```
  合成: 拼合 2/4 命中 | 合成「中断」「B05柜」| 总耗时 6.2s
```

整句回退时：

```
  合成: 整句 MOSS | 8.1s | 原因: 通用规则无槽位
```

**颜色建议**（终端 ListView 阶段）：

- `命中` / `拼合` → 绿  
- `合成中` / `MISS` → 黄  
- `失败` / `回退` → 红  

### 6.3 试听与告警的一致性

- 试听走与告警 **同一套** `TtsPlaybackPlanner`（开拼合则试听也拼合）。  
- 试听进度条阶段文案扩展：  
  - `查缓存 2/4` → `合成片段「联通」` → `拼合播放` → `完成 0.3s`  

### 6.4 与 MOSS 引擎区联动

| MOSS 加速 | 片段拼合 | CPU 并行 | 说明 |
|-----------|----------|----------|------|
| GPU | 开 | 隐藏 | 拼合收益仍大（降延迟） |
| CPU | 开 | 显示 | 拼合为主，流式为辅（见 MOSS 方案五-B） |
| 任意 | 关 | — | 始终整句 MOSS，缓存仅作离线预热实验 |

## 九、交互与日志原则

1. **所有耗时操作有进度**：预热、批量合成、历史分析。  
2. **所有缓存决策可追溯**：`tts_cache.log` + 终端摘要双写。  
3. **破坏性操作需确认**：清空缓存、删除热词、覆盖 index。  
4. **默认保守**：拼合默认 **开**；自动预热默认 **关**（避免首次启动长时间占用）。  
5. **引擎非 MOSS 时**：隐藏缓存卡与 Tab 内 MOSS 专用项，避免干扰 SAPI/讯飞用户。
