# 项目结构

> 文档版本：2026-06-23  
> 仓库根目录：`alarm_speech`（示例本地路径 `e:\alarm_speech`）

本文描述当前仓库的目录与关键文件；**规划中、尚未实现的目录**（如 `cache/`）已标注「待实现」。

## 一、目录总览

```
alarm_speech/
├── alarm_env/                 # 主程序虚拟环境（setup_alarm.ps1，不提交）
├── moss_env/                  # MOSS 专用虚拟环境（setup_moss.ps1，不提交）
├── third_party/
│   └── MOSS-TTS-Nano/         # MOSS 源码与 ONNX 模型（setup_moss.ps1 克隆/解压）
├── logs/                      # 运行日志（不提交）
│   ├── app.log                # 应用总日志
│   ├── error.log              # 异常堆栈
│   └── moss_daemon.log        # MOSS 守护进程
├── cache/                     # 【待实现】语音片段缓存（见 UI优化方案 v5 §四）
├── docs/                      # 项目文档（本目录）
│
├── flet_demo.py               # UI 入口（Flet 0.28）
├── api_client.py              # 监控平台告警 API
├── alarm_processor.py         # 告警清洗、Diff、播报规则
├── tts_engine.py              # TTS 调度（SAPI / MOSS / 讯飞）
├── tts_config.py              # 配置合并、MOSS/CUDA/CPU 线程预设
├── moss_tts.py                # MOSS 客户端（守护进程 + CLI 回退）
├── moss_daemon.py             # MOSS 常驻 HTTP 服务
├── moss_cuda_env.py           # Windows CUDA DLL 路径与真实探测
├── moss_cuda_probe.py         # CUDA 可用性探测脚本
├── xfyun_tts.py               # 讯飞 WebSocket TTS
├── app_logger.py              # 落盘日志初始化
│
├── test_sapi_tts.py           # SAPI 独立试听
├── test_moss_tts.py           # MOSS 独立试听
│
├── setup_all.ps1              # 一键安装（环境 + 模型，推荐）
├── download_moss_models.py      # MOSS 模型预下载脚本
├── setup.ps1                  # 一键安装（可选 -WithMoss）
├── setup_alarm.ps1            # 安装 alarm_env
├── setup_moss.ps1             # 安装 moss_env + MOSS 仓库
├── fix_moss_win.ps1           # Windows 下 MOSS 依赖版本锁定
├── fix_moss_gpu.ps1           # onnxruntime-gpu + NVIDIA wheel
├── run.ps1                    # 启动主程序
│
├── requirements.txt           # alarm_env 依赖
├── config.json                # 默认配置（可提交）
├── config.local.example.json  # 本地配置示例
├── config.local.json          # 本地密钥与覆盖（不提交）
├── .gitignore
└── README.md                  # 仓库入口说明
```

## 二、Python 模块关系

```
flet_demo.py
    ├── api_client.py          → 监控 REST API
    ├── alarm_processor.py     → 事件 Diff、话术规则
    ├── tts_engine.py          → 播报队列
    │       ├── xfyun_tts.py
    │       ├── moss_tts.py    → moss_daemon.py（moss_env）
    │       └── Windows SAPI
    ├── tts_config.py
    └── app_logger.py          → logs/
```

| 模块 | 说明 |
|------|------|
| `flet_demo.py` | 桌面 UI：连接、轮询、试听、引擎、播报策略、数据终端 |
| `api_client.py` | HOST/Token、`fetch_history`、`fetch_active_alarms` |
| `alarm_processor.py` | `AlarmTracker`、`event_to_speech`、端口表规则 |
| `tts_engine.py` | 引擎切换、MOSS GPU/CPU、试听阻塞、去重队列 |
| `moss_tts.py` | 拉起 `moss_daemon`、合成 WAV、失败回退 CLI |
| `moss_daemon.py` | FastAPI：`/health`、`/status`、`/synthesize` |
| `moss_cuda_env.py` | `setup_cuda_dll_paths`、`probe_cuda_ready` |
| `tts_config.py` | `config.json` + `config.local.json`、MOSS CPU 线程预设 |

## 三、脚本与虚拟环境

| 路径 | 用途 |
|------|------|
| `alarm_env/` | 主程序：Python 3.9+，Flet、requests 等 |
| `moss_env/` | MOSS：Python 3.12，onnxruntime-gpu、MOSS 依赖 |
| `run.ps1` | `alarm_env\Scripts\python.exe flet_demo.py` |
| `setup.ps1` | 调用 `setup_alarm.ps1`，可选 `-WithMoss` |
| `fix_moss_gpu.ps1` | GPU 加速安装与 `moss_cuda_probe.py` 验证 |

**原则**：主程序与 MOSS **环境隔离**；UI 通过子进程调用 `moss_env`，不在 `alarm_env` 内加载 ONNX。

## 五、MOSS 相关路径

| 路径 | 说明 |
|------|------|
| `third_party/MOSS-TTS-Nano/` | 上游仓库；含 `infer_onnx.py`、`onnx_tts_runtime.py` |
