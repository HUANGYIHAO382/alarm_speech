# 功能说明

本文档详细说明告警视听助手各功能模块的行为与使用场景。

---

## 一、连接配置

### 1.1 服务器 HOST

监控平台的基础地址，例如：`http://你的监控服务器地址`。

程序会向该地址发起 HTTP 请求，访问告警搜索接口：

```
GET {HOST}/api/monitor/alarm/search
```

### 1.2 访问令牌（Token）

通过 HTTP 请求头 `token` 字段传递，用于平台鉴权。Token 仅在界面中填写，不会写入代码或提交到 Git。

---

## 二、告警数据拉取

### 2.1 历史告警（100 条）

**操作**：点击「历史告警 (100条)」按钮。

**行为**：
- 按最近告警时间倒序，拉取最多 100 条全量记录（含已恢复与未处理）
- 在右侧「数据终端」中以格式化文本展示每条告警的：
  - 状态（已恢复 / 未处理）
  - 时间、节点、IP、事件标题、详情内容
- 结果会缓存到内存，供「测试语音」功能使用

### 2.2 活跃告警轮询

**操作**：选择轮询间隔（30 秒 / 1 分钟），点击「开始」进入轮询；再次点击变为「停止」。

**行为**：
- 后台线程按设定间隔自动拉取 **未处理且未忽略** 的活跃告警（最多 50 条）
- 与本地追踪池对比，产生事件并可选语音播报
- 终端实时展示：
  - **事件播报日志**（最新在上，最多保留 40 条）
  - **当前活跃告警列表**
- 左下角状态栏显示轮询状态与「追踪 N 条」徽章

---

## 三、状态追踪（Diffing）

由 `AlarmTracker` 类实现，是轮询模式的核心逻辑。

### 3.1 追踪池

程序在内存中维护一个 `known_alarms` 字典，以告警 `id` 为键，记录当前正在追踪的告警。

### 3.2 首轮预热

第一次轮询时，将当前活跃告警全部加入追踪池，**不播报**（避免启动时轰炸式语音）。

### 3.3 事件类型

| 事件 | 含义 | 是否语音播报 |
|------|------|-------------|
| **NEW** | 发现新的活跃告警 | 是 |
| **RECOVERED** | 告警已从活跃列表消失，且反查确认 `is_recover=1` | 是 |
| **DISMISSED** | 告警消失但非恢复（如被忽略、状态变更） | 否 |
| **STALE** | 反查超时或失败，暂保留追踪 | 否 |

### 3.4 恢复反查

当某条告警从活跃列表中消失时，程序会调用 `query_by_id` 按 ID 反查详情，以区分「真正恢复」与「其他原因消失」。恢复播报使用反查后的最新数据（含最新 `last(2)` 状态）。

---

## 四、播报规则

界面「播报规则」下拉框提供三种模式，用于将告警 dict 转换为适合 TTS 朗读的中文文本。

### 4.1 自动匹配（默认）

- 若告警标题或内容包含「端口表」相关关键字，且能解析 `last(2)=up/down`，则使用 **端口表规则**
- 否则使用 **通用规则**

### 4.2 端口表告警

专门针对端口状态类告警，从详情中提取 `last(2)` 的值：

| last(2) 值 | 播报中文 |
|------------|----------|
| up | 联通 |
| down | 中断 |

**播报格式示例**：

```
B05柜，NE20E杠1，端口，联通
```

设备名转换规则：`B05_NE20E-1` → `B05柜，NE20E杠1`（连字符读作「杠」）。

### 4.3 通用规则

- **未恢复**：`告警，{设备名}，{告警标题}`
- **已恢复**：`恢复，{设备名}，已恢复正常`
- 若实为端口表告警，仍会优先尝试端口表话术

### 4.4 文本润色

`polish_speech_for_tts` 会对文本做兜底处理：统一设备名格式、将英文 up/down 替换为中文、去除多余空格等，提升 TTS 朗读自然度。

## 五、语音播报（TTS）

### 5.1 语音引擎

| 引擎 | 说明 | 依赖 |
|------|------|------|
| **Windows SAPI 离线** | 使用系统自带中文语音包 | Windows、pywin32（可选） |
| **讯飞在线** | 科大讯飞 WebSocket 合成 | 网络、`config.local.json` 密钥 |

未配置讯飞密钥时，程序默认使用 Windows SAPI。

### 5.2 语音播报开关

「语音播报」开关控制轮询产生事件时是否实际发声。关闭后仍会在终端记录事件，但不播放语音。

### 5.3 SAPI 语速调节

选择 Windows SAPI 引擎时，可通过滑块调节语速（范围 -10 ~ 10，默认 -4，略慢于系统默认）。

### 5.4 试听功能

| 按钮 | 作用 |
|------|------|
| **试听 SAPI** | 强制使用 Windows SAPI 朗读「SAPI 试听文字」框中的内容，不受引擎下拉框和播报开关影响 |
| **测试语音** | 取最近一条历史告警（或自动拉取 1 条），按当前播报规则转换后，用当前选定引擎播报 |

### 5.5 播报队列与去重

- TTS 在后台线程中消费队列，避免阻塞 UI
- 相同文本在 3 秒去重窗口内不会重复播报

## 六、数据终端

右侧「数据终端」区域模拟运维控制台风格：

- 等宽字体（Consolas）、绿色文字、深色背景
- 展示历史查询结果、轮询状态、事件日志、语音测试详情
- 支持一键清空（右上角扫帚图标）

---

## 七、API 接口说明

`AlarmApiClient` 封装了监控平台告警搜索接口的三种用法：

| 方法 | 用途 | 主要参数 |
|------|------|----------|
| `fetch_history(limit)` | 全量历史，含已恢复 | `limit`：条数 |
| `fetch_active_alarms(limit)` | 仅活跃告警，用于轮询 | 过滤 `is_ignore=0` 且 `status=0` |
| `query_by_id(alarm_id)` | 按 ID 反查单条 | 用于确认恢复状态 |

统一返回 `ApiResult` 结构：

```python
ApiResult(ok: bool, data: list, error: str)
```

## 八、配置体系

配置按优先级合并（后者覆盖前者）：

1. `config.json`（可选，可提交仓库的默认配置）
2. `config.local.json`（本地私密配置，**不提交 Git**）
3. 环境变量（如 `XFYUN_APP_ID`、`XFYUN_API_KEY`、`XFYUN_API_SECRET`）

讯飞 TTS 典型配置结构：

```json
{
  "tts": {
    "provider": "xfyun",
    "xfyun": {
      "app_id": "你的APPID",
      "api_key": "你的APIKey",
      "api_secret": "你的APISecret",
      "vcn": "xiaoyan"
    }
  }
}
```
