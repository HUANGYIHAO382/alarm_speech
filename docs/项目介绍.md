# 项目介绍

## 项目名称

**告警视听助手**（仓库名：`alarm_speech`）

## 项目定位

本项目是一款面向网络/设备监控运维场景的 **告警拉取 + 语音播报** 桌面助手。它连接监控平台的告警 API，将告警信息以可视化终端形式展示，并在检测到新增告警或设备恢复时，自动转换为适合语音朗读的中文话术进行播报，帮助值班人员在不一直盯着屏幕的情况下及时获知异常。

## 适用场景

- 机房或网络运维值班，需要 **听告警** 而非只看屏幕
- 监控平台已提供告警搜索 API，希望通过 Token 快速接入
- 告警内容包含 **端口表状态**（如 `last(2)=up/down`）等结构化信息，需要定制播报话术
- Windows 环境下希望使用 **本地离线 TTS**，或可选接入 **科大讯飞在线语音**

## 技术栈

| 类别 | 技术 | 说明 |
|------|------|------|
| 语言 | Python 3.8+ | 主开发语言 |
| 桌面 UI | [Flet](https://flet.dev/) | 基于 Flutter 的 Python 桌面界面框架 |
| 网络请求 | requests | 调用监控平台 REST API |
| 语音合成（在线） | 科大讯飞 WebSocket TTS v2 | 需配置 API 密钥 |
| 语音合成（离线） | Windows SAPI | 通过 win32com / pyttsx3 / PowerShell 兜底 |
| 运行环境 | Windows | 本地 TTS 与 winsound 播放依赖 Windows |

> **说明**：本项目为 **Python + Flet** 桌面应用，并非 Web 前端项目。

## 核心能力概览

1. **告警数据拉取**：支持历史告警查询与活跃告警轮询
2. **状态追踪（Diffing）**：对比前后两次拉取结果，识别新增、恢复、解除追踪等事件
3. **播报规则引擎**：通用规则、端口表专用规则、自动匹配三种模式
4. **双 TTS 引擎**：Windows 本地 SAPI 与讯飞在线语音可切换
5. **深色运维风格 UI**：左侧控制面板 + 右侧类终端数据展示区

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    flet_demo.py（UI 入口）                    │
│  连接配置 / 轮询控制 / 播报规则 / TTS 设置 / 数据终端展示      │
└──────────────┬──────────────────┬─────────────────────────┘
               │                  │
       ┌───────▼───────┐  ┌───────▼────────┐  ┌──────────────┐
       │ api_client.py │  │alarm_processor │  │ tts_engine.py│
       │  告警 API 层   │  │ 清洗/追踪/规则  │  │  语音播报调度 │
       └───────┬───────┘  └────────────────┘  └──────┬───────┘
               │                                      │
               ▼                                      ▼
     监控平台 REST API                    xfyun_tts.py / Windows SAPI
     /api/monitor/alarm/search            tts_config.py（配置加载）
```

### 模块职责

| 文件 | 职责 |
|------|------|
| `flet_demo.py` | UI 入口：搭建 Flet 界面，响应用户操作，调度各模块，刷新终端输出 |
| `api_client.py` | 纯网络层：封装告警搜索 API，返回结构化 `ApiResult` |
| `alarm_processor.py` | 数据清洗、终端格式化、状态追踪、播报规则转换 |
| `tts_engine.py` | TTS 引擎调度：队列消费、去重、双引擎切换、语速控制 |
| `xfyun_tts.py` | 科大讯飞 WebSocket 流式语音合成客户端 |
| `tts_config.py` | 合并 `config.json`、`config.local.json` 与环境变量 |
| `test_sapi_tts.py` | Windows SAPI 语音试听测试脚本 |

## 设计原则

- **分层解耦**：API、业务处理、语音、UI 各自独立，API 变更时只需修改 `api_client.py`
- **敏感信息不入库**：HOST、Token、讯飞密钥均在本地配置，不写入代码仓库
- **播报可配置**：支持多种规则与引擎，适应不同告警类型
- **运维友好**：终端风格输出、事件日志、追踪数量徽章，便于值班观察
